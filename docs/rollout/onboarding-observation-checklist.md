# DevPulse — Onboarding Observation Checklist
## For first-user shadowed sessions (watch someone install live)

**Method:** Screen share or recording. Say nothing. Observe. Note every hesitation.

---

### Pre-install

- [ ] What OS / VS Code version?
- [ ] What are they currently working on? (signals their collection type)
- [ ] Do they have a Postman collection ready?
- [ ] How did they find DevPulse? (channel attribution)

---

### Install moment

- [ ] Do they find the extension immediately in the marketplace?
- [ ] Do they read the description before installing?
- [ ] Do they hesitate at "Sign in" or "API key" step? *(trust checkpoint)*
- [ ] Do they go to devpulse.in to get a key without being told?
- [ ] Time from install to first attempt to connect: ______

---

### Onboarding flow — 3 critical moments

**Moment 1: First view of the welcome panel**
- [ ] Do they immediately understand what to do?
- [ ] Do they read the feature list or skip straight to the key input?
- [ ] Any visible confusion about where the panel is?

**Moment 2: API key entry**
- [ ] Do they know their key needs to start with `dp_`?
- [ ] Any friction getting to devpulse.in to generate a key?
- [ ] "Connected as [email]" message — does it feel reassuring?

**Moment 3: First scan**
- [ ] Do they know how to import a collection without prompting?
- [ ] Any confusion about "select collection to scan" picker?
- [ ] What do they do while the scan runs?
- [ ] First reaction to findings: ______

---

### Findings review

- [ ] Do they understand severity grouping immediately?
- [ ] Do they click into a finding to read it?
- [ ] Do they notice the "Mark resolved" action?
- [ ] Any confusion about status (open / in-progress / resolved)?

---

### Trust signals to watch

- [ ] Do they visibly hesitate before pasting an API key?
- [ ] Do they ask about data privacy?
- [ ] Do they check the README or GitHub before trusting?

---

### Red flags to catch immediately

- [ ] Confused about where the DevPulse panel is
- [ ] Can't find their Postman collection file
- [ ] Scan completes but they don't see findings (missed the panel)
- [ ] "No findings" on first scan → do they think it's broken?

---

### End of session questions

1. "What just happened — in your own words?"
2. "What would you do next?"
3. "What confused you?"
4. "Would you use this again tomorrow?"
5. "Who else on your team should see this?"
